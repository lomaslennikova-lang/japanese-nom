# japanese-nom
solving Japanese nomograms
## 1. Goal / Мета проєкту
Мета проєкту — створити вебсайт для роботи з японськими номограмами: графічними логічними головоломками, у яких приховане зображення відновлюється за числовими підказками для рядків і стовпців.

Сайт має дозволяти користувачам розв’язувати готові номограми, а також завантажувати власні варіанти зображень у вигляді сітки із заповненими та порожніми клітинками.

Функція створення нових номограм має належати самому сайту. Наприклад, сайт може щоденно генерувати кілька нових номограм різного розміру та різної складності.

Після створення або завантаження номограми система повинна автоматично генерувати числові підказки та перевіряти, чи є така номограма придатною для гри.

Головна ідея проєкту — пропонувати користувачам заздалегідь перевірені номограми, які мають єдине правильне рішення, що відповідає оригінальній картинці.

Для автора цей проєкт також є першим навчальним проєктом у підході Vibe Coding: поступовому створенні програмного продукту за допомогою AI-інструментів, через формулювання ідей, вимог, функцій, логіки та подальшу роботу з кодом.

## 2. Target Users / Користувачі
Основні користувачі сайту:

- Люди, які люблять логічні ігри, головоломки, судоку, пазли та кросворди.
- Початківці, які хочуть навчитися розв’язувати японські номограми.
- Досвідчені гравці, які хочуть отримувати якісні та перевірені головоломки.
- Користувачі, які хочуть завантажувати власні картинки-сітки для перевірки.
- Автор проєкту, для якого цей сайт є першим навчальним проєктом у Vibe Coding.

## 3. Main Features / Основні функції
### 3.1. Ігрове поле
Сайт має показувати номограму у вигляді сітки. Кожна клітинка може мати кілька станів:

- порожня;
- зафарбована;
- позначена хрестиком.

Користувач може змінювати стан клітинки кліком.

### 3.2. Числові підказки
Для кожного рядка і кожного стовпця сайт має показувати числові підказки.

Ці числа вказують, які групи зафарбованих клітинок повинні бути в конкретному рядку або стовпці.

### 3.3. Розв’язування номограми
Користувач розв’язує головоломку, зафарбовуючи клітинки відповідно до числових підказок для рядків і стовпців.

Під час розв’язування сайт має відстежувати правильне розташування окремих груп зафарбованих клітинок по горизонталі та вертикалі.

Якщо окрема група клітинок у рядку або стовпці правильно відповідає певному числу в підказках, це число має позначатися як виконане. Наприклад, його можна виділяти жирним шрифтом, змінювати колір або закреслювати.

Сайт також має дозволяти користувачу перевірити, чи поточне рішення є правильним.

### 3.4. Автоматичне створення номограм сайтом
Сайт має мати функцію автоматичної генерації нових номограм.

Наприклад, система може щоденно створювати кілька нових головоломок:

- різного розміру;
- різної складності.

Після генерації кожна номограма повинна проходити автоматичну перевірку. Тільки після успішної перевірки вона може бути додана до списку доступних для гри.

### 3.5. Завантаження номограми користувачем
Користувач може завантажити або вручну задати власну картинку-сітку, де частина клітинок є заповненою, а частина порожньою.

Після завантаження сайт має:

- проаналізувати картинку;
- згенерувати числові підказки для рядків і стовпців;
- перевірити, чи можна за цими підказками відновити саме оригінальну картинку;
- перевірити, чи має головоломка лише одне правильне рішення;
- позначити номограму як придатну або непридатну для гри.
